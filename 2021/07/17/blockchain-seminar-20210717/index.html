Speaker: Zhou Yu(俞州)

Paper 1(How Effective are Smart Contract Analysis Tools? Evaluating Smart Contract Static Analysis Tools Using Bug Injection)

Static Analysis Tools

  • Symbolic Execution
    • Oyente
    • Securify
    • Mythril
    • Manticore
  • Pattern matching
    • SmartCheck
  • Static Single Assignment
    • Slither

Bug Injection

Challenging

  • 找个合适的插入位置
    • 编译不通过
  • 语义依赖
    • 时间戳(block.timestamp())（可以被矿工恶意利用）

Smart Contract

  • 数量 50
  • 类型
  • 代码长度
  • 插入方式
    • 插入完整代码(Inject full code snippets)
    • 弱化安全机制(Weakening security mechanisms)
    • 修改代码(Transform codes)

Bug Type

  • Reentrancy
    • 一个函数在同时多次调用
  • Timestamp dependency
    • 依赖时间戳，被 矿工恶意利用
  • Unhandled Send
  • Unhandled exceptions
    • 异常终止合约
  • Transaction ordering dependency(TOC)
    • 打包区块，对用户交易顺序改变
  • Integer overflow/underflow
  • Use of tx.origin
    • 合约相互调用时
    • tx.origin: 最初调用者
    • msg sender: 上一个调用者

Solidify Algorithm

  1. 确认可以编译通过，生成抽象语法树，找所有可插入位置
  • 插入完整代码(Inject full code snippets)
    • 一行代码
    • 代码块（位置与function有关）
  • 弱化安全机制(Weakening security mechanisms)
    • tx.origin
    • Integer overflow/underflow
    • delete checking
  • 修改代码(Transform codes)
  1. 插入漏洞
  2. 在静态分析工具上分析
  • 正确性（类型，位置）
  • 设定不同阈值，用来判断误报

Exploitability

  • TOD测试代价较大，没有测试

Q&A

  • 插入的漏洞和源代码没有关系

Paper 2(LAVA: Large-scale Automated Vulnerability Addition)

自动收集漏洞

Source

  • Accident
  • Search
  • Injection
  • Synthesis

Taint-based measures

  • 被利用的数据，DUA（Dead, Uncomplicated And Available data）
  • TCN
    • 数字越大，变量被计算次数越多
    • 0代表直接从输入来，我们认为更容易被控制
  • liveness
    • 评估输入数据
    • 输入被用在分支
    • 决定分支越多，对程序控制流越大，越好被利用

Attack point selection

  • 在DUA出现之后，可以利用DUA，就是一个可以利用对位置

Data-flow bug injection

  • 在DUA和attack point之间建立Data-flow

Finding DUA

  • LAVA 大概的流程

实验室工作

  • Vulnerability Detection Platform
  • 因为缺少数据，所以想自己插入漏洞
  • bytecode，source code 两种数据的不同
  • 跨平台
